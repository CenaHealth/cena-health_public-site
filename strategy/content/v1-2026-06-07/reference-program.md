**Timeline stages (per IA §7, locked structure):**

1. **Referral** — patient enters via partner's referral pathway
2. **Enrollment** — partner's program coordinator enrolls + consents

**Heading:** Quiet H2 (writer's call — e.g., *"Who staffs what"* or *"Staffing the program: partner roles, Cena substrate, shared work"*). Front-load load-bearing noun per voice.md §3. Anchor `#staffing-model` per IA §7.

**Role clusters (per IA §7, locked three-cluster structure):**

1. **Roles the partner staffs (indication-specific clinical authority)** — the partner-org owns these; substrate does not.
2. **Roles Cena substrate handles (indication-agnostic structural work)** — the substrate owns these; partner does not need to staff them.
3. **Shared work (joint roles + meetings)** — both sides participate.

**Specific roles within each cluster (per IA §7):**

- **Partner staffs:**
  - Clinical champion / Program Director (the role that owns the program inside the partner-institution)
  - Program coordinator (operational lead — enrollments, scheduling, partner-side workflow)
  - Dietitian (clinical nutrition assessment + counseling)
  - Patient navigators (patient-facing, partner-employed)
  - (Indication-specific clinical roles — e.g., HIV-specialist physician for the live program; would change to e.g., endocrinologist for a diabetes program)
- **Cena substrate handles:**
  - Audit-log assembly (continuous; replaces half-FTE-of-spreadsheet-assembly per Phase 1 brief §5)
  - Adherence monitoring (continuous, automated; flags clinical staff when intervention threshold crosses)
  - Outcome attribution (continuous, per substrate's outcome-capture instrumentation)
  - Second-system clinical checks (continuous; per Phase 1 Appendix C #5)
- **Shared work:**
